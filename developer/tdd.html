My thoughts on Test Driven Development

I feel the three main points of Test Driven Development are

  • We write tests so that we can refactor code
  • We should let tests define our architecture
  • Use tests to reveal the code.

Don't use mocking frameworks

I dislike the terms Unit and Integration tests. They are technical definitions and people are really stuck to them. I prefer the more abstract Logic and API tests

A test should only test one thing. Usually. Does not mean it has one assert. Just define the thing properly.

Tests should have no dependencies.

Use in-memory dbs in API tests.

Most bugs appear when we map data

Let functionality drive data models (another entry?)
